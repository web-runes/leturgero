This guide gets you from zero to a running Vite Deploy dev server. Pick [New project](#new-project) to scaffold a starter for your host, or [Existing project](#existing-project) if you already have a Vite app.

## New project

<Steps>

1. Download a basic template for your favorite host:

    <Tabs>
      <TabItem label="Cloudflare" icon="cloudflare">
        ```sh
        npx giget@latest gh:web-runes/vite-deploy/examples/cloudflare-server
        ```
      </TabItem>
      <TabItem label="Netlify" icon="netlify">
        ```sh
        npx giget@latest gh:web-runes/vite-deploy/examples/netlify-server
        ```
      </TabItem>
      <TabItem label="Node" icon="node">
        ```sh
        npx giget@latest gh:web-runes/vite-deploy/examples/node-server
        ```
      </TabItem>
      <TabItem label="Vercel" icon="vercel">
        ```sh
        npx giget@latest gh:web-runes/vite-deploy/examples/vercel-server
        ```
      </TabItem>
    </Tabs>

2. Install dependencies:

    <PackageManagerTabs>
      <Fragment slot="npm">
        ```sh
        npm install
        ```
      </Fragment>
      <Fragment slot="pnpm">
        ```sh
        pnpm install
        ```
      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn install
        ```
      </Fragment>
    </PackageManagerTabs>

3. Start the development server:

    <PackageManagerTabs>
      <Fragment slot="npm">
        ```sh
        npm run dev
        ```
      </Fragment>
      <Fragment slot="pnpm">
        ```sh
        pnpm dev
        ```
      </Fragment>
      <Fragment slot="yarn">
        ```sh
        yarn dev
        ```
      </Fragment>
    </PackageManagerTabs>

</Steps>

## Next steps

Once your dev server is running:

- Pick an [output mode](/outputs/) (static, server, or hybrid) depending on whether you want SSG, SSR, or both.
- Follow the [deployment guide](/deploy/) for your host to ship to production.
- Browse the [integration guides](/integrate-with/) to wire Vite Deploy up with Hono, Express, or TanStack Start.
